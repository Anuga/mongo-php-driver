\MongoDB

\MongoDBManager

Manager abstracts a cluster of Server objects (i.e. socket connections).

Typically, users will connect to a cluster using a URI, and the Manager will perform tasks such as replica set discovery and create the necessary Server objects. That said, it is also possible to create a Manager with an arbitrary collection of Server objects using the static factory method (this can be useful for testing or administration).

Operation methods do not take socket-level options (e.g. socketTimeoutMS). Those options should be specified during construction.
